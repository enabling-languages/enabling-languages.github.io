The Multicultural Victoria Act 2011 states that all individuals in Victoria are equally entitled to access opportunities and participate in and contribute to the social, cultural, economic and political life of the state. Availability of information online translated into languages other than English is important to ensuring this is achieved.

Government departments and agencies have a responsability to ensure people with limited English, and people who are Deaf or hard of hearing, are given information in their own language to participate in decisions that affect their lives.

Context

Internationalisation is about the architecture of a site, in the first instance, then secondly about the markup and styling. It is not about content per se.

A small handful of sites targeting a foreign audience may localise their sites and translate most, if not all, their content.

This is not the norm. Most agencies and departments carefully select languages and content.

Overview

Multilingual Information Online does not exist in isolation, it is intended to supplement:

  • Digital standards for Victoria,
  • Using interpreting services, and
  • Effective translations.

Overview

The MIO guidelines equips digital teams across state government to improve access to, and quality of, online content in community languages.

  • Improve discoverability and accessibility of community language content.
  • Provide a baseline for Victorian Government departments and agencies.
  • Content covers the basics, linking out to more detailed resources.
  • Two audiences:
    1. Content owners
    2. Digital teams

Principles

  • Content should be inclusive, responsive, accessible and discoverable.
  • Use HTML in preference to other file formats.
  • Use Unicode (UTF-8).
  • Follow internationalisation best practice.
  • Be responsive to requirements of community languages.

Web internationalisation (i18n) – the basic techniques

The essential techniques (declaring an encoding, identifying language and change of language and bidirectional markup) were introduced in HTML 4.0, published on December 18, 1997.

An early multilingual example within the vic.gov.au domain from 1999.

Character encodings

The three basic rules:

  1. Use Unicode and UTF-8
  2. Only use Unicode
  3. No psuedo-Unicode, just Unicode!

Language tagging

The lang global attribute indicates the language of the element's text. The attribute contains a entry value in the format defined in the Tags for Identifying Languages (BCP47) IETF document.

Language tagging is required in order to provide good internationalisation, accessibility and discoverability.

WCAG 2.0 Success Criterion 3.1.1 – the default human language of each Web page can be programmatically determined. (Level A)

WCAG 2.0 Success Criterion 3.1.2 – the human language of each passage or phrase in the content can be programmatically determined except for proper names, technical terms, words of indeterminate language, and words or phrases that have become part of the vernacular of the immediately surrounding text. (Level AA)

The anatomy of a language tag

BCP 47 (RFC 5646 and RFC 4647)

language-extlang-script-region-variant-extension-privateuse

  • en-AU-simple
  • zh-Hans-CN
  • zh-Hans
  • ja
  • es-419
  • pt-BR

Choosing a language tag

  • Ensure you have the right language
  • Avoid collections
  • Use macrolanguages with care
    ku ☞ ckb, kmr, sdh
  • Avoid depreciated subtags
    i.e he instead of iw
  • where possible, usa a single language subtag, ie yue instead of zh-yue
  • Don't automatically use script subtags for non-textual content
  • Check for suppress script fields, ie en rather than en-Latn
  • Opt for the shorted meaningful language tag

Indicating the document language

To set the language of a document, add the lang attribute to the html element.

<html lang="din">
	⋮
</html>

This works best if the site is localised, or the CMS tracks the language of unique content and of the UI, and the templates inject a lang attribute value as required.

A work around

Setting the document language on the html is required to meet WCAG 2.0 Success Criteria 3.1.1. But this may not be possible on an existing website, a hack would be to wrap the unique content in a div and apply the lang attribute. But this does not clearly identify the language of the document.


<html lang="en-AU">
  <head>
    ⋮
  </head>
  <body>
    ⋮
    <div id="translatedContent" lang="din">
      ⋮
    </div>
    ⋮
  </body>
<html>

Nesting languages

The value of the lang attributes applies to both the content of the element and the values of attributes of the element.

Assuming you want an English language tooltip for the link, and the nav element inherits the lang value en-AU:

    <li><a href="/am" title="Amharic">

Text direction

The dir global attribute indicates the directionality of the element's text. It can have the following values:

  1. ltr – left to right
  2. rtl – right to left
  3. auto

auto, uses a basic algorithm as it parses the characters inside the element until it finds a character with a strong directionality, then applies that directionality to the whole element.

Setting the direction of document

To set the direction of a document, add the dir attribute to the html element.


<html lang="aii-Syrn" dir="rtl">
    ⋮
<html>

This works best if site is localised, or the CMS tracks the direction of unique content and of the UI, and the templates inject a dir attribute value as required.

A work around

Setting the overall direction on the html element is not always possible, a hack would be to wrap the unique content in a div and apply the dir attribute.

Strategies

Content management systems

  • Document level metadata
  • Translation content management/workflow
  • Template and themes:
    • Dynamic assignment of document language and direction
    • Theme should support most required LCG script languages
    • Language specific overrides
    • Web fonts for languages that require it, esp. mobile platforms.

Content management systems – editing interface

  • Source code mode is often useful for adding translated content.
  • Ensure editor filters do not strip lang and dir attributes.
  • Modify editor to allow markup of English language content with a translation.
  • Add mechanism for easily adding bidi control characters.

Language specific web typography

Default styling of HTML elements is based on the typographic and typesetting ocnventions of Western Europe. These may not be suitable for all the languages do deal with.

Language-related pseudo-classes and attribute selectors

  Psuedo class selector Attribute selector
Language :lang(ar) [lang="ar"]
Direction :dir(rtl) [dir="rtl"]

Javascript equivalents:

The HTMLElement.lang property gets or sets the base language of an element's attribute values and text content. See MDN.

The HTMLElement.dir property gets or sets the text writing directionality of the content of the current element. See MDN.

What to look out for

The obvious ones are: the font stack, font sizes and leading (line height).

Less obvious are ordered and unordered list markers, italicisation.

What to look out for – bidi

When supporting languages written LTR and RTL, it is neccessary to double-check any CSS property that can contain direction sensitive values. This is espacially important for fully localised sites.

E.g. background, background-image, background-position, background-position-x, border-bottom-left-radius, border-bottom-right-radius, border-color, border-left, border-left-color, border-left-style, border-left-width, border-radius, border-right, border-right-color, border-right-style, border-right-width, border-style, border-top-left-radius, border-top-right-radius, border-width, box-shadow, clear, cursor, direction, float, left, margin, margin-left, margin-right, padding, padding-left, padding-right, right, text-align, text-shadow, transform, transform-origin, transition

@counter-style myanmar-paren {
  system: override myanmar;
  prefix: "(";
  suffix: ")";
}
@counter-style hyphen {
  system: cyclic;
